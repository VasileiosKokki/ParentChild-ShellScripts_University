Λειτουργικά Συστήματα : 2η Εργασία

προγραμματισμός συστήματος σε C

Άσκηση 2η (25 μονάδες)

Σκοπός της εργασίας είναι να δημιουργήσετε το δικό σας πρόγραμμα που θα διαχειρίζεται την πρόσβαση και την ανάγνωση/εγγραφή σε ένα αρχείο. Οι κανόνες λειτουργίας του περιγράφονται παρακάτω.

Όταν θα ξεκινάει το πρόγραμμα θα πρέπει να δημιουργεί δύο διεργασίες (μέσω της fork)

  • Η μία διεργασία θα είναι υπεύθυνη για την αρχική εγγραφή στο αρχείο καθώς και για το γενικο συγχρονισμό της εκτέλεσης (πχ περιμένοντας τη διεργασία αναγνωστών να τελειώσει μέσω της waitpid). Η αρχική εγγραφή να βασίζεται στην εγγραφή ενός string το οποίο θα αποτελείται από 2000 random χαρακτήρες a-z σε ένα αρχείο data.txt
  • Η δεύτερη διεργασία θα είναι υπεύθυνη για την ανάγνωση από το αρχείο και την εξαγωγή των πόσων και ποιων χαρακτήρων έχει από την αλφάβητο, αφού έχει τελειώσει η εγγραφή από την πρώτη διεργασία. Η ανάγνωση θα πρέπει να παραλληλοποιείται μέσω πολλαπλών νημάτων (4 νήματα) χρησιμοποιώντας την Pthreads βιβλιοθήκη. Κάθε νήμα θα πρέπει να αναλαμβάνει την ανάγνωση ενός κομματιού (2000/4 = 500 χαρακτήρες) από το αρχείο (με τη βοήθεια της lseek) και να κρατάει τοπικά τα δικά του αποτελέσματα για την εμφάνιση κάθε γράμματος. Στο τέλος της επεξεργασίας θα πρέπει να φροντίζει να προσθέτει τα δικά του στοιχεία στην κεντρική δομή αποθήκευσης. Η δεύτερη διεργασία θα πρέπει επίσης να εκτυπώνει τα περιεχόμενα της κεντρικής δομής, δηλαδή πόσες εμφανίσεις έχει κάθε χαρακτήρας από το a έως το z (δεν μας ενδιαφέρουν οι υπόλοιποι ascii χαρακτήρες όπως !,%κλπ καθώς και τα κεφαλαία), πριν το τέλος της.  
  • Οι δύο διεργασίες θα πρέπει να συγχρονίζουν τη σειρά εκτέλεσης μέσω σημαφόρων ώστε να διασφαλίζεται ότι πάντα θα ξεκινάει η 1η διεργασία και θα υπάρχει περιεχόμενο στο αρχείο. Η 2η διεργασία θα πρέπει να ξεκινάει μόνο εφόσον έχει τελειώσει το μέρος εγγραφής από την 1η. Η 1η διεργασία δεν επανεγγράφει σε κάποια μεταγενέστερη χρονική στιγμή το αρχείο, μόνο στο αρχικό στάδιο..
  • Αν ληφθεί σήμα SIGINT ή SIGTERM να ρωτάει το χρήστη αν όντως θέλει να τερματίσει το πρόγραμμα, και αν ναι, τότε να το τερματίζει.

Χρήσιμες κλήσεις συστήματος:

  • fork(), exec(), open(), close(), read(), write(), dup(), dup2(), lseek()
  • pthread_create(), pthread_join(),pthread_mutex_lock/unlock
  • wait(), waitpid()
  • signal()

Μεθοδολογία Υλοποίησης:

  • Προσοχή στη σειρά εκτέλεσης. Πρώτα θα πρέπει να εκτελείται η διεργασία εγγραφής και στη συνέχεια η ανάγνωσης. Αλλά μία διεργασία θα πρέπει να έχει το ρόλο του συντονιστή και να περιμένει να τελειώσει η άλλη μέσω της waitpid.
  • Προσοχή στο συγχρονισμό μέσω σημαφόρων των 2 διεργασιών. Πού πρέπει να δηλώνεται ο σημαφόρος για να είναι προσβάσιμος και γνωστός και στις 2 διεργασίες; Σε τι τιμή θα πρέπει να αρχικοποιείται και τι πράξεις να κάνει πάνω του κάθε διεργασία;
  • Χρειάζεται να διαμοιράζονται αυτές οι δύο διεργασίες κάποιον file descriptor; Δικαιολογήστε την απάντησή σας
  • Χρειάζεται να μοιράζονται όλα τα νήματα της 2ης διεργασίας-αναγνώστη κάποιον file descriptor ανάγνωσης; Χρειάζεται προστασία αυτός από ταυτόχρονη πρόσβαση; Δικαιολογήστε την απάντησή σας
  • Για την αποθήκευση της μέτρησης των ascii characters, αρκεί να χρησιμοποιήσετε ένα πίνακα με integers, σε κάθε θέση του οποίου θα προσθέτετε +1 κάθε φορά ανάλογα με την αριθμητική τιμή του χαρακτήρα (αφού αφαιρέσετε το αντίστοιχο ASCII offset). Π.χ το a θα πρέπει να προσθέτει +1 στο array[0], το b στο array[1] κλπ. Ένα παράδειγμα type casting για να παίρνετε την αριθμητική τιμή ενός χαρακτήρα ακολουθεί:  
  • char c=’r’;
  • int b=(int) c;
  • Προσοχή στην ταυτόχρονη πρόσβαση στον κεντρικό πίνακα της μέτρησης. Είναι κρίσιμη περιοχή; Χρειάζεται προστασία; Δικαιολογήστε την απάντησή σας
  • Φροντίστε να μην υπάρχουν διεργασίες zombie
  • Για τη δημιουργία των random χαρακτήρων μπορείτε να χρησιμοποιήσετε το /dev/urandom ή να το υλοποιήσετε με το δικό σας τρόπο στη c. 
  • Φροντίστε το αρχικό αρχείο να έχει 2000 χαρακτήρες μόνο a-z, ώστε να μπορείτε να κάνετε επιβεβαίωση της ορθής λειτουργίας του προγράμματος στην τελική εκτύπωση (π.χ ότι όντως μέτρησε 2000 χαρακτήρες a-z αθροίζοντας όλες τις θέσεις του πίνακα)
  • Μπορείτε να υλοποιήσετε τμήματα της εργασίας σε περίπτωση που δυσκολευτείτε, π.χ. Να έχετε δεδομένο το αρχείο με το text και να υλοποιήσετε μόνο τη διεργασία με τα νήματα και το συγχρονισμό τους. Η βαθμολόγηση θα είναι αντίστοιχη της υλοποίησης.

Παρατηρήσεις

  • Οι κατάλογοι θα έχουν δημιουργηθεί αυτόματα για όσους το είχαν δηλώσει για την 1η Εργασία
  • Τα εκτελέσιμα αρχεία θα πρέπει να εκτελούνται σε λειτουργικό σύστημα  Linux.
  • Να παραδώσετε την εργασία σας μέσα από το e-class μέχρι την Δευτέρα 30 Ιανουαρίου και ώρα 12 τα μεσάνυχτα.
  • Για να είναι έγκυρη η υποβολή σας θα πρέπει να παραδώσετε:
  • Την αναφορά σας στο eclass. Η αναφορά σας θα πρέπει να έχει συγκεκριμένη μορφή. Για το λόγο αυτό μπορείτε να χρησιμοποιήσετε: το google doc template “2η Εργασία Λειτουργικά 2022-2023” το οποίο θα βρείτε στο https://docs.google.com/document/u/0/?ftv=1&tgif=d  αν κάνετε Login με το account του σας στο χαροκόπειο
  • Ένα αρχείο tar.gz το οποίο θα αντιγράψετε στον linux server σε συγκεκριμένο κατάλογο (οδηγίες ακολουθούν)
  • Η αναφορά να είναι ΥΠΟΧΡΕΩΤΙΚΑ σε μορφή pdf και το όνομά του να είναι της μορφής username_e2.pdf δηλαδή αν ο Α.Μ. σας είναι it202101 το αρχείο σας θα είναι it202101_e2.pdf - ΑΥΤΟ ΘΑ ΑΝΕΒΑΣΕΤΕ ΣΤΟ ECLASS
  • Φροντίστε να τεκμηριώνετε τις απαντήσεις σας και η αναφορά σας να είναι ευανάγνωστη και δομημένη σωστά και να αναφέρει τα στοιχεία σας στο εξώφυλλο. Θα υπάρχει ποινή στον τελικό βαθμό για αναφορές που δεν τηρούν τους κανόνες. Σε κάθε περίπτωση θα πρέπει να έχετε ενδεικτικές εκτελέσεις (screenshots από το terminal σας)
  • Αν θέλετε μπορεί να έχετε και κάποιο makefile ή οποιοδήποτε αρχείο είναι απαραίτητο για τo compilation/εκτέλεση του προγράμματός σας και να αναφέρετε τις οδηγίες στην αναφορά σας
  • Για να είναι έγκυρη η εργασία σας θα πρέπει να ετοιμάσετε ένα tar.gz αρχείο το οποίο θα περιέχει τα εξής:

it202101.c
it202101_report.pdf

Η εγκυρότητα του αρχείου θα ελεγχθεί με κάποιο shell script συνεπώς αν τα αρχεία σας δεν συμμορφώνονται με τον παραπάνω κανόνα ΔΕΝ ΘΑ ΓΙΝΟΝΤΑΙ ΑΠΟΔΕΚΤΑ.

Πως να δοκιμάσετε αν είναι έγκυρο το αρχείο που θα υποβάλετε:

η εντολή η οποία θα χρησιμοποιηθεί για να ελέγξει τη δομή του tar.gz αρχείου θα είναι η εξής:

tar -tzvf it202101.tar.gz  αν το αρχείο σας είναι το it202101.tar.gz και θα πρέπει να σας εμφανίζει το αποτέλεσμα:

-rw-r--r-- it202101/it202101             0 2017-11-10 13:41 it202101.c
-rw-r--r-- it202101/it202101             0 2017-11-10 13:41 it202101_report.pdf

Για να δημιουργήσετε το αρχείο tar.gz θα συγκεντρώσετε όλα τα αρχεία σας σε έναν κατάλογο και τρέξτε την εντολή tar -cvzf it202101.tar.gz *.c *.pdf

Που θα υποβάλετε τα αρχεία σας

Στον server με ip 10.100.59.103 έχει δημιουργηθεί ένας κατάλογος /users/os/ex2 ο οποίος περιέχει έναν κατάλογο για καθένα από εσάς. Δηλαδή αν είστε ο χρήστης it202101 θα αντιγράψετε το αρχείο που δημιουργήσατε πιο πάνω και θα ονομάζεται it202101.tar.gz στον κατάλογο /users/os/ex2/it202101

Για να βεβαιωθείτε ότι το αρχείο σας έχει αντιγραφεί σε αυτόν τον κατάλογο θα τρέξετε την εντολή

 ls - l /users/os/ex2/it202101

 -rw-r--r-- 1 it202101 it202101 0 Jan 10 14:03 /users/os/ex2/it202101/it202101.tar.gz
